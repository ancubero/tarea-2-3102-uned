# Configuración del Workflow
Los siguientes son los pasos para configurar un Workflow para desplegar un sitio web

* Ingresar a github.com
* Seleccionar el repositorio en el que trabajaremos
* Una vez en el repositorio dar click sobre "Settings"
* Bajo se sección "Code and automation" dar click sobre "Pages"
* Buscar en la pagina el titulo "Build and deployment"
* Bajo la sección "Source" seleccionar "Deploy from a branch"
* Bajo la sección "Branch" seleccionar el branch deseado y luego el directorio deseado
* Por último dar click sobre el boton de "save"

# Problematica
Con la implementación de esta automatización, se logra resolver la problematica que ocurre cuando queremos actualizar un sitio web, ya que muchas veces por algun descuido o falta de información por parte de la persona que realizó la modificación, no se transmitio toda la información acerca de cuales fueron los archivos modificado, dando como resultado que dicha actualización pueda romper la forma en que el sitio se muestre.
